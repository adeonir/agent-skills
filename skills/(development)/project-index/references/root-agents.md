# Root AGENTS.md

Generate or update the `AGENTS.md` file at the project root.

## When to Use

Auto-loaded by overview.md and summary.md after generating their outputs. Not a direct trigger.

## Workflow

### Step 1: Collect Available Content

Check which `.agents/` files exist and extract key content:

| Source | Section | Required |
|--------|---------|----------|
| `.agents/project.md` | Project summary | Yes (skip if missing) |
| `.agents/codebase/conventions.md` | Key conventions | If exists |
| `.agents/codebase/commands.md` | Important commands | If exists |
| `.agents/codebase/checklist.md` | Post-task checklist | If exists |

### Step 2: Check Existing AGENTS.md

If `AGENTS.md` already exists at project root:
- Identify manually added sections (not generated by project-index)
- Preserve manual additions
- Update only generated sections

### Step 3: Generate Content

Structure:

```markdown
# {Project Name}

{One-paragraph project summary from project.md}

## Conventions

{Key conventions table from conventions.md -- top entries only}

## Commands

{Essential commands from commands.md -- setup, dev, test, lint}

## Checklist

{Post-task checklist from checklist.md}

## References

- `.agents/project.md` -- project context
- `.agents/codebase/` -- full codebase analysis
```

### Step 4: Write File

Write `AGENTS.md` at the project root.

## Size Guidelines

- Target: ~60 lines, max 100 lines
- AGENTS.md is a quick reference, not a full document
- Link to `.agents/` files for details instead of duplicating content
- Conventions: max 5 most important rows
- Commands: max 10 most used commands
- Checklist: keep as-is (already concise)

## Guidelines

**DO:**
- Keep it scannable -- this is the first file an agent reads
- Preserve manual additions when updating
- Maintain one file, one purpose: quick project orientation

**DON'T:**
- Duplicate full content from `.agents/` files
- Remove manually added sections when regenerating
