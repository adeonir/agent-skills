# Root AGENTS.md

Generate, update, or skip the `AGENTS.md` file at the project root based on
which agent-instruction convention the project already uses.

## When to Use

Auto-loaded by overview.md and summary.md after generating their outputs. Not a direct trigger.

## Workflow

### Step 1: Detect Root Files

Check which agent-instruction files exist at the project root:

- `AGENTS.md` -- cross-tool convention, project-index territory
- `CLAUDE.md` -- Claude Code convention, user-authored territory

### Step 2: Decide Action

| State | Action |
|-------|--------|
| Only `AGENTS.md` exists | Update it (Steps 3-6) |
| Only `CLAUDE.md` exists | Skip AGENTS.md generation. Report: "CLAUDE.md detected at root -- project-index will not create AGENTS.md to avoid duplicating agent instructions. If you want agents to pick up `.agents/` context automatically, add a pointer to `.agents/project.md` manually in CLAUDE.md." |
| Both exist | Update `AGENTS.md` only (Steps 3-6). Never touch `CLAUDE.md`. |
| Neither exists | Ask: "No agent-instruction file found at root. Create `AGENTS.md` for quick orientation? (y/n)". If `y`, run Steps 3-6. If `n`, skip. |

Record the outcome (`created`, `updated`, `skipped`) so the caller (initialize.md / summary.md) can mirror it in its report.

### Step 3: Collect Available Content

Check which `.agents/` files exist and extract key content:

| Source | Section | Required |
|--------|---------|----------|
| `.agents/project.md` | Project summary | Yes (skip if missing) |
| `.agents/codebase/conventions.md` | Key conventions | If exists |
| `.agents/codebase/commands.md` | Important commands | If exists |
| `.agents/codebase/checklist.md` | Post-task checklist | If exists |

### Step 4: Check Existing AGENTS.md

If `AGENTS.md` already exists at project root:
- Identify manually added sections (not generated by project-index)
- Preserve manual additions
- Update only generated sections

### Step 5: Generate Content

Structure:

```markdown
# {Project Name}

{One-paragraph project summary from project.md}

## Conventions

{Key conventions table from conventions.md -- top entries only}

## Commands

{Essential commands from commands.md -- setup, dev, test, lint}

## Checklist

{Post-task checklist from checklist.md}

## References

- `.agents/project.md` -- project context
- `.agents/codebase/` -- full codebase analysis
```

### Step 6: Write File

Write `AGENTS.md` at the project root.

## Size Guidelines

- Target: ~60 lines, max 100 lines
- AGENTS.md is a quick reference, not a full document
- Link to `.agents/` files for details instead of duplicating content
- Conventions: max 5 most important rows
- Commands: max 10 most used commands
- Checklist: keep as-is (already concise)

## Guidelines

**DO:**
- Detect root state before writing -- respect whichever convention the project already uses
- Keep AGENTS.md scannable -- this is the first file an agent reads
- Preserve manual additions when updating
- Maintain one file, one purpose: quick project orientation
- Return the outcome (`created` / `updated` / `skipped`) so the caller can report it

**DON'T:**
- Edit or overwrite `CLAUDE.md` -- it is user-authored territory
- Create `AGENTS.md` when `CLAUDE.md` is the only root file without asking the user
- Duplicate full content from `.agents/` files
- Remove manually added sections when regenerating

## Error Handling

- `.agents/project.md` missing: cannot generate AGENTS.md meaningfully -- skip and inform user to run initialize first
- User declines the "neither exists" prompt: skip silently, record outcome as `skipped (user declined)`
